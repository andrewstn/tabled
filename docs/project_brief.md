# Tabled Project Brief

Tabled is a production-minded Ruby on Rails application for student organizations and small teams.

It helps student organization officers manage members, roles, events, attendance, announcements, and internal operations.

The app should feel warm, campus-native, practical, and robust — not like a generic AI/B2B SaaS dashboard.

## Core idea

Student organizations need a shared place to manage the real operational work that happens across a semester: members, officers, events, RSVPs, attendance, announcements, and participation history.

## Primary users

- Student organization officers
- Volunteer coordinators
- Event chairs
- Project team leads
- Club members
- Residence hall committees

## Product personality

Tabled should feel like a digital version of:

- a shared meeting table
- a club office
- a sign-in sheet
- a campus bulletin board
- a well-organized student org binder

Avoid sterile SaaS styling, AI-style gradients, generic metric cards, and corporate productivity language.

## Visual direction

Use:

- warm neutrals
- soft paper-like surfaces
- readable typography
- tactile buttons
- calm forms
- useful empty states
- understated shadows and borders
- practical layouts
- specific campus/org language

Avoid:

- sterile gray/blue SaaS dashboards
- purple AI glow effects
- glassmorphism
- excessive gradients
- vague “optimize your workflow” copy
- generic “all-in-one platform” positioning

## MVP

Milestone 1: Organization foundation

- Users can sign up, log in, and log out.
- Users can create organizations.
- Users can belong to multiple organizations through memberships.
- Memberships have roles: owner, officer, coordinator, member.
- Organization data is securely scoped.
- Members can view organization dashboards.
- Owners and officers can access settings.
- App has local seed data.

Future milestones:

- Invitations
- Member directory
- Events
- RSVPs
- Attendance and check-ins
- Announcements
- Activity log
- CSV import/export
- Recruiter-facing demo workspace
